# AIC9 温控 ADRC：标定、数字孪生与开表控制

对 AIC9 加热炉（通道 1，PLC 10ms 周期）的 ADRC（自抗扰）温度控制项目：
真机 bump 标定执行器特性 → 数字孪生辅助设计 → 上机拨盘收尾。

**当前状态：第一代（代码改动 [1]~[9]）生产就绪**——100→400°C 到达 1.5~1.7s / 首超 ≤3°（暖机典型日），
控制器内无 if-else，现场只有 lrKd 一个旋钮、随配方下发。[10] 刹车预算律已认证、带价签封存（默认关）。

## 二、如何使用（按场景找入口）

### 场景 1：新机器从零部署（主入口）
打开 `analysis/adrc_gen1_guide.html`，按开头"作战地图"的 7 步走，约 1.5 个工作日。
数字全部重测，流程与代码原样复用。安全红线（500°C 上限、450 中止线、每步峰值预算）在指南第0步，先读后动。

### 场景 2：日常生产
配方表带 lrKd 列自动下发（400 类 0.85 / 440 类 1.05 / 200 类 1.1），操作工零操作。两条规矩：
①冷机第一炉先热机一次；②阶跃前起点稳 ≥15s。
现场唯一旋钮：首超 >4° → 该配方 kd +0.05；不碰线爬行 → kd −0.05。

### 场景 3：换硬件 / 老化复标
指南第6步"换硬件重标范围"表：换加热元件半天；换功率调节器先 15 分钟甄别场（保温 MV 带 / 断电初降速 / 90% 与 22% 斜率对照）再定范围；换整机全流程重走。
老化仪表：稳态锚点（保温 MV ↔ 真保温功率）漂移 <10% 不动；10~20% 重标 q 表；>20% 全重标。

### 场景 4：拿到真机 CSV 后的判读命令

```bash
python3 analysis/theta_from_bump.py 数据.csv               # 粗判: θ/q断点/稳态锚点 → 可粘贴 ST 变量表
python3 analysis/theta_from_bump.py 数据.csv --write-twin  # 确认无误后写入孪生
python3 analysis/fit_taue.py 数据.csv                      # 精判: 自动找手动跳变, 整段拟合分离 θ/τe/q∞
python3 analysis/replay_identify.py 数据.csv 跳变时刻s      # 回放辨识: 确认机上实跑参数与标注一致
python3 analysis/test_drift_tolerance.py                   # 判读脚本自检(合成数据)
```

前提：CSV 列顺序 time/PV1/PV2/SV/MV（按位置认列）、采样 10ms 级。
新机器第一遍注意 h1/c 自举坑（指南第2步 2.2 埋坑表第二行）。
`fit_qinf_0715.py` 是 7-15 场的参考实现（路径与跳变时刻写死），新数据一律用 `fit_taue.py`。

### 场景 5：改性能目标 / 设计新档位（只有这时才需要仿真）

复现第一代不需要仿真（指南第2步有说明）。要设计新档位时走完整闭环：

1. 标定（指南第1~2步）→ 2. 更新孪生 `twin_params.json`（改前备份）→
3. 仿真扫描（`analysis/ladder_v2_scan.py` 式，**标称合格 + 应力可接受**才算候选：增益 ×0.85/×1.15、θ+0.03、τ2×1.3）→
4. 上机按档位爬梯，每档一发 100→400 存 CSV → 5. `replay_identify.py` 复核 + 指标对账，偏差 >20% 回第 2 步。

孪生快速上手：

```python
import sys; sys.path.insert(0, 'analysis')
from twin import Twin, metrics
tw = Twin()                                   # 读 analysis/twin_params.json
yy, uu = tw.closed_loop_adrc(b0=3.4, wc=2.0, wo=9.0, wr=3.0, nd=28,
                             y0=100.0, svt=400.0, T=20.0)
print(metrics(yy, 100.0, 400.0))              # (rise90[s], 超调[°C], settle±1°[s])
```

开表后的 v 空间闭环（含 lrTauF/lrKd）参考 `ladder_v2_scan.py` 的 `cl()` 实现。

## 四、已知边界（不是待改进——是物理）

- **硬定理**：任何日子任何状态都"1.5s + ≤3°"没有控制器能承诺（冲刺前测不到的状态 × 只热不冷的单向执行器 = 尾部风险）。第一代承诺：暖机典型日达标，坏日子多 2~5°，失败方向可控。
- **200 靶点 + 热炉体**：墙温高于靶温时免费刹车变负，首超 5°+ 是物理不是整定。
- **物理地板**：暖机 100→400 理想开关极限 ~1.5s 且随炉况浮动 1.50~1.61s（元件蓄热滑行刹不掉）；1.71s 距地板仅 0.2s。
- **设备上限**：500°C 瞬时可触不可驻留（>480°C ≤1min）；各测试场中止线 450。

完整诚实条款见指南参考D；历史结论（含被证伪的五轮结构假设）全程留痕在各分析脚本文件头。
