系统辨识 —— FOPDT 模型与拟合方法

从真实开环脉冲测试数据，到 K/τ/L 三个数，再到能直接用的 PI 参数：用的什么模型、怎么拟合出来的、哪些段能信、拿到参数之后怎么整定——以及"自衡还是积分型"的模型结构之争怎么用数据裁决。

01

原理

拿真实记录的输入信号去驱动一个假设的模型，看仿真曲线和实测曲线像不像，不像就调参数——是曲线拟合，不是图解公式。

FOPDT（First-Order Plus Dead Time，一阶惯性+纯滞后）是用三个数描述"输入变化后，输出怎么响应"的简化模型：

FOPDT 模型
τ·dy/dt = K·u(t-L) - y
K：稳态增益（最终输出变化 ÷ 输入变化）　τ：时间常数（多快逼近最终值）　L：死区（纯延迟）

为什么是这个模型，不是别的

这是自平衡对象（self-regulating，下文简称"自衡"）专用的模型——固定输入，输出最终会稳定在一个值上，不会一直涨。这是数据里直接能看到的事实：维持 300℃ 需要持续 15-18% 的 MV（输入不为零而温度恒定，说明散热在实时抵消输入），断电后温度还会自己降回来。对"积分型模型是不是更贴近实际"这个合理质疑的完整数据裁决，见 §05。

这一点值得强调，是因为经典教材（如 Åström & Hägglund《PID Controllers》）里另有一套积分型对象（integrating process）的辨识方法，模型是 G(s)=K/[s(1+sT)]·e^(-sL)——这种对象固定输入后输出会一直匀速爬升，不会自己稳定。这套方法配的是"图上读渐近线交点、代公式解T"的图解法，跟我们这里用的模型和方法都不是一回事，用错了会文不对题。

怎么拟合——仿真法非线性最小二乘，不是图解法

具体步骤：

  1. 取真实记录的一段 MV(t)、PV(t)，减去脉冲前的基线，得到相对偏差量 u(t)、y_meas(t)
  2. 给定候选 (K, τ, L)，把 u(t) 整体平移 L（模拟死区），再用前向欧拉法数值积分 FOPDT 方程，得到仿真曲线 y_sim(t)
  3. 定义残差 = y_sim(t) − y_meas(t)，交给 scipy.optimize.least_squares 自动迭代调整 (K, τ, L)，让残差平方和最小

这是拿整条曲线做回归，不是像教材图解法那样只读一两个特征点代公式——对噪声更不敏感，因为误差会被整条曲线的信息平均掉。

✓ 拟合代码本身验证过

担心"代码会不会本身有 bug"，先拿人工构造的、已知真实 K/τ/L 的合成数据喂给这套拟合流程——结果几乎精确地把设定的真值找回来（R²=1.0000）。这一步只验证"拟合方法本身没问题"，不代表真实数据每一段都能拟合出干净的结果（见 §03）。

02

代码实现

实际产出 §03 那张结果表的完整脚本。

数据清洗 —— load()

原始记录的时间戳有重复、顺序也不一定严格递增，直接积分会报错——先去重、按时间排序、转成相对秒数。

正向仿真 —— simulate_fopdt()

死区用插值实现：把输入信号在时间轴上整体平移 L，相当于"L秒之前的输入，此刻才生效"。每个真实采样间隔内部再切成更小的子步（子步长 ≤ τ/5），前向欧拉积分，避免采样间隔相对 τ 太粗导致数值不准。

拟合 —— fit_fopdt()

残差 = 仿真曲线 − 实测曲线，交给非线性最小二乘求解器；R² 是拟合完成后额外算的，用来判断这一段的拟合质量好不好（见 §03 哪些能信）。

sysid_fopdt3.py
import numpy as np
import pandas as pd
from scipy.interpolate import interp1d
from scipy.optimize import least_squares

def load(fname, sheet):
    df = pd.read_excel(fname, sheet_name=sheet, engine='openpyxl')
    df.columns = ['t','pv1','pv2','sv','mv']
    df['t'] = pd.to_datetime(df['t'])
    df['sec'] = (df['t'] - df['t'].iloc[0]).dt.total_seconds()
    df = df.drop_duplicates(subset='sec').sort_values('sec').reset_index(drop=True)
    return df

def simulate_fopdt(t, u, K, tau, L, y0):
    u_interp = interp1d(t, u, bounds_error=False, fill_value=(u[0], u[-1]))
    ud = u_interp(t - L)
    y = np.empty_like(t, dtype=float)
    y[0] = y0
    for i in range(1, len(t)):
        dt = t[i] - t[i-1]
        n_sub = max(1, int(np.ceil(dt / (tau/5 + 1e-9))))
        yi = y[i-1]
        for k in range(n_sub):
            yi = yi + (dt/n_sub) * (K*ud[i] - yi) / tau
        y[i] = yi
    return y

def fit_fopdt(t, mv, pv, mv_base, pv_base, p0, bounds):
    u = mv - mv_base
    y_meas = pv - pv_base
    def resid(params):
        K, tau, L = params
        return simulate_fopdt(t, u, K, tau, L, y0=0.0) - y_meas
    res = least_squares(resid, p0, bounds=bounds, xtol=1e-12, ftol=1e-12, max_nfev=8000)
    pred = simulate_fopdt(t, u, *res.x, y0=0.0) + pv_base
    ss_res = np.sum((pv-pred)**2); ss_tot = np.sum((pv-pv.mean())**2)
    r2 = 1 - ss_res/ss_tot
    return res.x, r2, pred

⚠ 这批结果仍没有覆盖到的地方
  • 都是在~99℃基线附近做的脉冲——真正需要的100-400℃核心工作区间，这批数据只能算旁证，不是直接证据。
  • 这批数据没有直接给出静态增益曲线g(u)——后续ADRC/MPC设计里用的g(u)曲线是另外构造的近似版本（只标定了2-3个点），不是从这批数据严格拟合出来的，细节见analysis/adrc_reference.html里的说明。而且 §05 的平衡 MV 数据表明 g(u) 本身还会随热历史漂移 ±2-3%MV——静态曲线只能当"大致先验"，不能当精确真值。

要把这些缺口补上，需要的是之前反复提到的同一件事：多个功率点、每点保持足够久（≥5τ，也就是65秒以上）、覆盖实际使用的100-400℃范围的开环测试——这一步目前还没做。

04

SIMC 整定

有了 K、τ、L，怎么系统性地算出 PI 参数——解析公式，不是试凑。

SIMC（Skogestad Internal Model Control）是从内模控制理论推出来的解析整定公式：给定 §03 辨识出来的 FOPDT 模型 (K, τ, L)，直接算出 PI 控制器的 Kc、Ti，不需要现场试凑。

核心设计思想

假设被控对象真的精确等于辨识出来的 FOPDT 模型，选一组 Kc、Ti，使闭环之后的响应恰好退化成一个干净的一阶响应，时间常数是 τc——这是唯一自己挑的设计参数，"想要多快"就填多小。Kc、Ti 就是从这个要求反解出来的：

SIMC 整定公式
Kc = (1/K)·(τ/(τc+L)) Ti = min(τ, 4·(τc+L))
τc：期望闭环时间常数，自己选，越小越激进

Ti 公式里那个 min() 在做什么

  • 追求快（τc 很小）：Ti≈4(τc+L)，被死区主导，积分追得急
  • 不那么激进（τc 接近或超过 τ/4）：Ti 被 τ 本身封顶——这是经典的零极点对消：PI 的零点(1/Ti)抵消对象的极点(1/τ)，Ti=τ 时对消最干净

simc_tuning.py
"""
SIMC (Skogestad Internal Model Control) tuning: turn an identified FOPDT model
(K, tau, L -- from sysid_fopdt3.py) into PI parameters (Kc, Ti), by choosing one
design parameter tau_c (desired closed-loop time constant) instead of tuning
Kc/Ti by trial and error.

Design idea: choose Kc, Ti such that IF the plant exactly matched the FOPDT
model, the closed loop would collapse to a clean first-order response with
time constant tau_c. The formulas below are the algebraic result of that
requirement (see Skogestad, "Simple analytic rules for model reduction and
PID controller tuning", 2003).
"""

def simc_pi(K, tau, L, tau_c):
    Kc = (1.0 / K) * (tau / (tau_c + L))
    Ti = min(tau, 4 * (tau_c + L))
    return Kc, Ti

if __name__ == '__main__':
    K, TAU, L = 16.94, 13.0, 0.2   # from sysid_fopdt3.py, K at u=15%
    print(f"{'tau_c':>6s} {'Kc':>8s} {'Ti':>8s}")
    for tc in [0.2, 0.3, 0.5, 1.0, 2.0, 5.0]:
        kc, ti = simc_pi(K, TAU, L, tc)
        print(f"{tc:6.1f} {kc:8.3f} {ti:8.2f}")
⚠ SIMC 解决的是"给定一个 K"，不是"K 会变"这件事

SIMC 公式假设 K 是个固定数——但真实执行器的 K 从 12.5 变到 30.9（§03 已经测出来）。SIMC 本身不处理这个变化，算出来的 Kc/Ti 只对拿去算它的那个 K 附近最合适。

试过把这条也线性化（让 K 在公式里恒等于 1，PID 输出再过一遍 g⁻¹ 反解，跟 ADRC 线性化是同一个思路）——结果不是更好，是更差：线性化后超调从 1.7-5.6% 涨到 12% 左右。原因是 PID 的积分+抗饱和逻辑假设"积分量和最终输出大致线性对应"，插入非线性反解之后这个假设被破坏了。ADRC 的 ESO 不依赖这个假设，所以同样的线性化对它有效，对 PID 反而帮倒忙——细节见 analysis/adrc_reference.html。

05

自衡，还是积分型？

一个合理的质疑：这类加热对象会不会其实更像"一阶惯性＋积分＋纯滞后"？把两种模型摆到同一段实测数据上，让数据裁决。

先回顾 §01 的那对术语：自衡＝恒定输入下输出最终稳住（散热随温度升高而增大，最终与输入打平）；积分型（integrating，无自衡）＝恒定输入下输出匀速爬升不止，只有输入为零才停。竞争的两个模型结构：

两种候选模型
A 自衡 FOPDT： τ·dy/dt = K·u(t−L) − y B 积分＋惯性＋滞后： G(s) = K′·e^(−Ls) / [ s·(1+τ₂s) ]
B 的 K′ 是斜率增益（℃/s 每 %MV）：输入恒定为 u 时，输出以 K′·u 的速度匀速爬升；τ₂ 是加热丝自身热起来的快惯性

证据一：断电后温度自己往下掉

60% 脉冲断电后，PV 从 363℃ 以最快 −27℃/s 自由下坠。积分型模型在输入归零后预测温度原地保持——它没有散热项。把 B 在上升段拟合到近乎完美（R²=0.999）再外推到降温段，R² 直接掉到 −12.6；A 一条曲线拟合全窗口，R²=0.986：

左：全窗口（1.1s 脉冲＋自由降温），B 断电后水平冻结；右：只看上升段，两个模型几乎重合

300℃18.4 → 16.7 → 15.2%三次到访依次递减——周边越烧越热，维持功率越低
400℃21.8 → 20.6%两次到访，同样的漂移方向

积分型对象下，任何非零的恒定输入都会让温度无限爬升——恒温只可能对应零输入。持续供 15~22% 功率而温度纹丝不动，说明有散热在实时抵消输入：这就是自衡。顺带注意平衡 MV 随热历史漂移 ±2~3%（同一个 300℃，三次数字不同）——静态曲线 g(u) 本身会移动，这正是 ADRC 的 ESO 需要实时兜底、纯查表前馈不够用的直接证据。

但在闭环的时间尺度上，两种说法等价

τ≈12~13s，而闭环一次升温动作只有 1.5~3.5s。时间远小于 τ 时，自衡对象的阶跃响应 1−e^(−t/τ) ≈ t/τ——就是一个带延迟的积分器。上图右半是定量版本：上升段 A、B 都是 R²≈0.99，而且两者对"斜率增益"的估计完全一致——A 的 K/τ = 3.487，B 拟合出的 K′ = 3.487（℃/s 每 %MV）。分歧只在 10 秒以上的自由降温／长保温段显现，而闭环从不让系统自由跑那么久。

更实际的是：两种模型代进 SIMC，算出来的控制器一个字都不差。

SIMC 在两种模型下给出同一组参数
自衡： Kc = (1/K)·(τ/(τc+L)) Ti = min(τ, 4(τc+L)) 积分型： Kc = (1/K′)·(1/(τc+L)) Ti = 4(τc+L) 代入 K′ = K/τ：两个 Kc 逐字相等； 只要调得快（4(τc+L) < τ，这里是 2.0s < 13s），两个 Ti 也相等。
§04 那组 Kc=1.535 / Ti=2.00 用哪个模型算都一样——Ti 里的 min() 本质就是"调得快时自动把对象当积分器处理"

ADRC 那边更直接：ESO 的内部模型本来就是 dy/dt = b0·u + f——一个积分器加"总扰动"，而 b0 = K/τ = K′ 恰好是两种模型达成一致的那个数。所以这场模型之争对已完成的 PID/ADRC 整定没有任何影响；真正需要自衡假设的只有两处：预测"温度最终稳在哪"（静态曲线 g(u) 只对自衡对象存在），以及 10 秒以上的长保温行为。

selfreg_vs_integrating.py
"""
Adjudicate the model-structure question with real burst data:

  A: self-regulating FOPDT           tau*dy/dt = K*u(t-L) - y
  B: integrator + fast lag + delay   G(s) = K'*e^(-Ls) / [s*(1+tau2*s)]

Two hard facts decide it (write-up: analysis/sysid_reference.html section 05):
  1. Free decay. After the 60% burst is cut, PV falls at up to -27 C/s. An
     integrator predicts PV holds level once input returns to zero -- B fitted
     on the rise alone (R2=0.999) collapses to R2=-12.6 when extrapolated over
     the decay; A fits the whole window at R2=0.986.
  2. Balance power. Holding 100/300/400 C steady takes ~2.5 / 15-18 / 21-22 %MV
     (closed-loop file plateaus, printed below). An integrating process can
     hold a constant level only at ZERO input.

Equally important: on the 1-3 s timescale the closed loop actually lives on,
the two models agree exactly -- A's K/tau = 3.487 = B's fitted K' (C/s per
%MV) -- and SIMC yields identical Kc/Ti from either model (the fast-tuning
branch of the Ti min()). The distinction only matters for long holds and for
predicting where PV settles (a static map g(u) exists only if self-regulating).

Also documents the window-contamination fix: the original windows in
sysid_fopdt3.py start exactly when the PID engages, so ~20 s of closed-loop
2-3% MV hold sits inside the "open-loop" fit -- dragging K low and L high
(hence the old L=0.47-0.71 s vs the 0.19-0.21 s seen in closed-loop data).
This script trims to burst+decay only [1206.5, 1260] (burst 1207.1, cut
1208.3): the clean 60% fit gives K=41.3, tau=11.9 s, L=0.32 s -- consistent
with the L=0.2 s used in the final controller designs.

Run from the repo root. Writes plots/06_自衡vs积分_模型对比.png.
"""
import numpy as np
import pandas as pd
from scipy.interpolate import interp1d
from scipy.optimize import least_squares
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt

F_OPEN   = 'AIC9_DATA-20260702-185858_203502-数据分析.xlsx'
F_CLOSED = 'AIC9_DATA-20260702-203502_204340-PID闭环数据.xlsx'

def load(fname, sheet):
    df = pd.read_excel(fname, sheet_name=sheet, engine='openpyxl')
    df.columns = ['t','pv1','pv2','sv','mv']
    df['t'] = pd.to_datetime(df['t'])
    df['sec'] = (df['t'] - df['t'].iloc[0]).dt.total_seconds()
    df = df.drop_duplicates(subset='sec').sort_values('sec').reset_index(drop=True)
    return df

def sim_fopdt(t, u, K, tau, L):
    ud = interp1d(t, u, bounds_error=False, fill_value=(u[0], u[-1]))(t - L)
    y = np.zeros_like(t)
    for i in range(1, len(t)):
        dt = t[i]-t[i-1]; n = max(1, int(np.ceil(dt/(tau/5+1e-9))))
        yi = y[i-1]
        for _ in range(n):
            yi += (dt/n) * (K*ud[i] - yi) / tau
        y[i] = yi
    return y

def sim_intlag(t, u, Kp, tau2, L):
    # tau2*dv/dt = u(t-L) - v ;  dy/dt = Kp*v   (integrator driven through a fast lag)
    ud = interp1d(t, u, bounds_error=False, fill_value=(u[0], u[-1]))(t - L)
    y = np.zeros_like(t); v = 0.0
    for i in range(1, len(t)):
        dt = t[i]-t[i-1]; n = max(1, int(np.ceil(dt/(tau2/5+1e-9))))
        yi = y[i-1]
        for _ in range(n):
            v  += (dt/n) * (ud[i] - v) / tau2
            yi += (dt/n) * Kp * v
        y[i] = yi
    return y

def fit(simfn, t, u, ymeas, p0s, lo, hi):
    best = None
    for p0 in p0s:   # multistart: single-start least_squares can stall in a local minimum here
        r = least_squares(lambda p: simfn(t, u, *p) - ymeas, p0,
                          bounds=(lo, hi), xtol=1e-12, ftol=1e-12, max_nfev=8000)
        if best is None or r.cost < best.cost:
            best = r
    pred = simfn(t, u, *best.x)
    ss_res = np.sum((ymeas-pred)**2); ss_tot = np.sum((ymeas-ymeas.mean())**2)
    return best.x, 1-ss_res/ss_tot, pred

# ---- evidence 2: balance MV at each settled plateau of the closed-loop file ----
print('closed-loop plateaus (balance MV = mean MV over the settled last 30%):')
dc = load(F_CLOSED, 'data')
sv = dc['sv'].values
change = np.where(np.diff(sv) != 0)[0]
starts = np.r_[0, change+1]; ends = np.r_[change, len(sv)-1]
for s, e in zip(starts, ends):
    if e - s < 100: continue
    tail = dc.iloc[s + int(0.7*(e-s)) : e]
    err = (tail['pv1'] - sv[s]).abs().mean()
    if sv[s] > 0 and err < 0.5:
        print(f'  SV={sv[s]:5.0f}C  t={dc["sec"].iloc[s]:6.1f}..{dc["sec"].iloc[e]:6.1f}s'
              f'  balance MV={tail["mv"].mean():6.2f}%  (|PV-SV|={err:.2f}C)')

# ---- evidence 1 + timescale equivalence: fit both models on the 60% burst ----
d = load(F_OPEN, 'data')
W0, W1 = 1206.5, 1260.0          # burst at 1207.10, MV cut at 1208.32
seg = d[(d['sec']>=W0)&(d['sec']<=W1)].reset_index(drop=True)
t = seg['sec'].values - W0
u = seg['mv'].values              # absolute MV (balance ~2.5% at 100C is worth ~1C; ignored)
y = seg['pv1'].values - 100.0     # PID had PV pinned at 100.00C before the burst

pA, r2A, prA = fit(sim_fopdt, t, u, y,
                   [[30,13,0.3],[8,10,1.0],[25,9,0.2],[15,20,0.5]],
                   [0.1,1,0], [60,60,5])

mrise = t <= 3.0                  # rise-only fit for B: where an integrating model lives
pB, r2Brise, _ = fit(sim_intlag, t[mrise], u[mrise], y[mrise],
                     [[2.5,0.3,0.1],[4,0.5,0.0],[1.5,0.1,0.3],[3,1.0,0.2]],
                     [0.01,0.03,0], [20,10,5])
prB = sim_intlag(t, u, *pB)
r2Bfull = 1 - np.sum((y-prB)**2)/np.sum((y-y.mean())**2)

print(f'\nA self-reg FOPDT (full-window fit): K={pA[0]:.2f} C/%MV  tau={pA[1]:.2f}s  L={pA[2]:.3f}s'
      f'  R2(full)={r2A:.4f}   K/tau={pA[0]/pA[1]:.3f} C/s/%MV')
print(f"B integrator+lag (rise-only fit):   K'={pB[0]:.3f} C/s/%MV  tau2={pB[1]:.3f}s  L={pB[2]:.3f}s"
      f'  R2(rise)={r2Brise:.4f}  R2(full, extrapolated)={r2Bfull:.4f}')

# ---- figure ----
fig, axes = plt.subplots(1, 2, figsize=(13, 4.8))
ax = axes[0]
ax.plot(t, y, '.', ms=2.5, color='#999', label='measured (60% burst, then MV=0)')
ax.plot(t, prA, '-',  lw=1.8, color='#1c66b0',
        label=f'A self-reg FOPDT, full fit  R$^2$={r2A:.2f}')
ax.plot(t, prB, '--', lw=1.8, color='#c1670a',
        label=f'B integ.+lag, rise fit -> extrapolated  R$^2$={r2Bfull:.2f}')
ax.axhline(0, color='#ccc', lw=0.8)
ax.set_title('full window: 1.1 s burst + free decay')
ax.set_xlabel('t (s)'); ax.set_ylabel(u'PV − 100 (°C)')
ax.grid(alpha=0.3); ax.legend(fontsize=9, loc='upper right')
i30 = np.searchsorted(t, 30)
ax.annotate('B: power off -> integrator holds level',
            xy=(30, prB[i30]), xytext=(24, prB[i30]+45), fontsize=9, color='#c1670a',
            arrowprops=dict(arrowstyle='->', color='#c1670a'))
ax.annotate('measured: falls at up to 27 °C/s\n= heat loss = self-regulating',
            xy=(30, y[i30]), xytext=(33, y[i30]+80), fontsize=9, color='#555',
            arrowprops=dict(arrowstyle='->', color='#888'))

ax = axes[1]
m = t <= 4.0
ax.plot(t[m], y[m], 'o', ms=4, color='#999', mfc='none', label='measured')
ax.plot(t[m], prA[m], '-',  lw=1.8, color='#1c66b0', label=f'A: K={pA[0]:.1f}, τ={pA[1]:.1f}s, L={pA[2]:.2f}s')
ax.plot(t[m], prB[m], '--', lw=1.8, color='#c1670a', label=f"B: K'={pB[0]:.2f} °C/s/%MV, τ₂={pB[1]:.2f}s, L={pB[2]:.2f}s")
ax.axvspan(1207.10-W0, 1208.32-W0, color='#c1670a', alpha=0.08)
ax.text(1207.7-W0, -30, 'MV=60%', fontsize=8, color='#a55', ha='center')
ax.set_title('zoom on the rise: the ~1-3 s timescale the loop lives on')
ax.set_xlabel('t (s)'); ax.set_ylabel(u'PV − 100 (°C)')
ax.grid(alpha=0.3); ax.legend(fontsize=9, loc='lower right')
fig.tight_layout()
out = 'plots/06_自衡vs积分_模型对比.png'
fig.savefig(out, dpi=110)
print('plot ->', out)
