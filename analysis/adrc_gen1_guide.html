0.1 术语速查（本文自造/行话词汇，先读这张表）

词白话解释
bump 场"bump"=踢一脚。在稳定温度上，手动把 MV 一步跳到某个值、保持几秒、跳回来——从温度的反应里读出炉子的"油门特性"。整场就是按不同油门深度各踢一脚
q(u) 表 / q∞ 表
(代码里的 aMapU/aMapV)		油门标定表：油门踩到 u%（aMapU 列），炉温最终以多少 °C/s 上升（aMapV 列）。"∞"表示取等加热元件烧透之后的稳态速率——不是踢完前半秒的表观速率（那个偏低，因为元件还没烧起来）
θ 纯滞后
(代码里的 nDeadSteps)		从"改 MV"到"温度计开始有反应"之间的死时间。控制周期 10ms，θ=0.12s 就填 nDeadSteps=12
τe 元件惯性
(代码里的 lrTauF)		加热元件从通电到火力全开的爬升时间（AIC9 约 0.24s）。它决定了"松油门后还有多少存热刹不住"
稳态锚点在某温度稳住 ≥60s 时的保温 MV 读数（如"保 400°C 要 18%"）。它是免费的炉况仪表：墙热了读数变小、设备老化读数变大
首超第一次冲过目标温度的峰值超出量（℃）。本系统超调后单调回落，不振荡，所以"超调预算"只花在首超上
冲透 / 爬行到达目标的两种形态：冲透=带着 1~3° 首超干脆利落碰线（快）；爬行=无限逼近但迟迟不触线（慢但零超调）
悬崖lrKd 只差 0.05，结果就在冲透/爬行之间翻面的那个位置。生产值要选在悬崖的安全一侧留一格
免费刹车断电后炉子靠散热自己降温的能力。它随目标温度和墙温变化 5 倍以上——这就是 lrKd 要按目标温度分档的物理原因
拨盘场给 lrKd 定值的机上流程。名字来源：现场只拧这一个旋钮，按一条单调规则（超了加、爬了减）像拨收音机刻度盘一样收敛，故名
回放辨识把控制器的复制品喂真机录下的数据，核对它算出的 MV 是否与实录一致——用来确认"机器上真的在跑你以为的那组参数"（抓过两次不一致）
z2ESO 观测器的"总扰动"估计：所有没建模的东西（散热、日漂、开门、老化）的实时总账。稳态时 −z2 = 当前保温功率
安全先行（第 1 天纸面完成，血的教训）：开工前书面确认三件事—— ①设备温度上限与驻留规则（AIC9：500°C 瞬时可触不可驻留，>480°C 勿超 1 分钟）； ②测试中止线（本指南各场取 450°C：任何时刻 PV≥450 立即 MV=0）； ③每张操作卡的每一步都必须附"峰值温度预算"（起点 + 电平×时长×加热速率 ≤ 中止线）， 预算算不过的步骤不许下发。

2.1 操作序列

#动作这一步之后你得到什么峰值预算
1开录 CSV（10ms 全程不断）。若炉子已停机 ≥2h，先静置录 10min环境温度 amb（静置段 PV 均值直读）—
2闭环稳 300°C ≥60s，抄下此时的保温 MV稳态锚点①：300°C ↔ 保温 MV%（老化监控的第一个基准读数）300
3切手动冻结当前 MV，只停 3~5 秒，然后一步跳到 12%，保持 8 秒，跳回原 MV，回闭环再稳 ≥30s油门表的第一个点：aMapU=12 那一行的 aMapV 值（由这 8 秒的升温曲线整段拟合得出）；同一段曲线的起动延迟给 θ、爬升快慢给 τe12% 是最小电平、升温最慢（AIC9 上 8s 远够安全）；新机器首发仍按下方爬梯法 ≤2s 试探后再放满
4换电平重复步骤 3，每次都从 300°C 基线出发（下列时长是 AIC9 实例）：
22% ×6s ／ 30% ×4s ／ 40% ×3s ／ 55% ×2s ／ 70% ×1.5s ／ 90% ×1s		每踢一脚 = 油门表再添一个点；全部踢完 aMapU/aMapV 齐了。90% 那发同时给出高温缩水（lrKqHot）的判读素材每步算过 ≤440。时长不是通用常数——它们是用 AIC9 已测出的速率反推的；换机器必须按下方"陌生机器安全爬梯法"逐级现场推导，禁止照抄
5收尾：闭环 SV=430 稳 ≥60s 抄保温 MV，然后 SV=0 断电，记录仪多录 ≥10min 自由降温再停稳态锚点②（高温端）；断电后头 3 秒的降速 = 该保温 MV 的真实功率（诚实性检查）；10min 降温段留档（将来升级用）430
规程红线：①手动段只允许"冻结→3~5 秒→一步跳变"——真机开环拿不住恒温， 禁止在手动模式下长时间"等稳"；②任何阶跃前在起点闭环稳 ≥15 秒（"台阶叠发"会让元件带着预热起跑，数据不可比）； ③PV≥450 立即 MV=0。
陌生机器安全爬梯法——步骤 4 的时长必须现场推导，不能照抄
上表"22%×6s … 90%×1s"是用 AIC9 已经测出来的各电平速率反推的预算。这里有个先后矛盾： 预算需要速率，而速率恰恰是这一场要测的东西。换一台热质量小或元件猛的炉子， 同样的 90%×1s 可能不是 +25° 而是 +100° 以上。所以在陌生机器上，时长要弱先强后、逐级外推地现场算出来：
  1. 试探发：全场第一脚只允许最小电平（12%）× 最多 2 秒，提刀规则（见③）全程生效。 这一发实测到的速率是后面一切预算的种子。若 12% 就冲出 >10°C/s，说明这台炉子远比 AIC9 猛： 基线降到 200°C、后续时长全部减半，重新开始爬。
  2. 逐级外推：已测电平 u₁ 实测速率 r₁，则下一电平 u₂ 的保守预估速率取 r₂预估 = r₁ × (u₂/u₁) × 2，安全时长 t₂ ≤ (中止线 − 起点)/r₂预估 − 1s（那 1 秒留给滑行，见③）。 ×2 安全系数的来由：功率调节器（相角控制类）中段常呈超线性，电平翻倍功率可能不止翻倍。 测完 u₂ 拿到真 r₂，再算下一级。禁止跳级（相邻电平比不超过 ~1.5，外推才可信）。
  3. 提刀规则（与预算并行，永远生效）：踢之前算好提刀温度 = 中止线 − 当前电平预估速率 × 1s； 盯着 PV，到提刀温度立即 MV=0，不管卡上时长走没走完。切断后温度还会因 θ+τe 的元件存热再顶一小截 （"滑行"），这 1 秒余量就是留给它的。
按此法爬完一遍后，你手里就有了这台机器自己的速率表——下次复标就可以像上表一样直接写死时长了。

为什么 kd 按配方下发而不是一个数：kd 补的是"理论阻尼需求 − 炉子自带的免费刹车"， 而免费刹车 = 耦合系数 × (腔温 − 墙温)，从 200°C 靶点到 440°C 靶点变化 5 倍以上。 这不是控制器缺陷，是把"每类工况的刹车预算"作为工艺数据放进配方——与商用温控器"多组 PID 参数"同机制， 但每组只有一个数、且有单调拨盘规则。系统本来就按配方下发 SV，kd 只是同表多一列，控制器内无任何 if-else。

4. 理论：行为参数的初值从哪来

把已测的滞后并入，环路近似二阶系统，特征方程 τ·s² + (1+kd)·s + wc = 0，其中 τ = θ + τe（AIC9 v1: 0.36s）：

设计量公式AIC9 数值例
自然频率（决定到达快慢）ωn = √(wc/τ)√(4.5/0.36) ≈ 3.5 rad/s → 300° 阶跃穿越 ~1.5s
阻尼比（决定超调）ζ = (1+kd)/(2√(wc·τ))目标 ζ≈0.8~0.85 → kd 理论初值 ≈ 1.0~1.1
wr（参考快慢）wr ≫ ωn 即为"阶跃参考"30（再大无区别；wr 小时 TD 平滑本身兼职阻尼，就回到低速档形态）
wo（观测器带宽）(2~4)×ωn 带内取10

kd 实测值（0.85~1.1）落在理论初值（1.0~1.1）与孪生折价值之间——因为炉子自带的免费刹车抵了一部分理论阻尼需求， 而免费刹车随靶点/炉态变。所以理论只给初值和方向，最终数值必须由场C 在机器上收尾——这是"认证仪器是机器"原则。

为什么去掉 SV 平滑后必须有 [9]：wr 小的时候参考慢慢走，环路准静态工作，平滑在兼职做阻尼； wr=30 后控制律退化为"比例 + 0.36s 执行器滞后"= 欠阻尼二阶，wc 越大 ζ 越小 （wc=1.55 首超 ~8°，wc=4 无阻尼可荡 40°）。lrKd 用 ESO 的速度估计 v̂ = b0·vd + z2 补回阻尼—— 这就是 PID 里 D 项的角色，但 v̂ 来自模型不对 PV 求导，无噪声放大。

5. 场B + 场C：上机爬梯与拨盘

5.1 场B：第1档回归（15 分钟，烧录后的看门关卡）

参数: lrWc=1.0  lrWo=12  lrWr=3.0  nDeadSteps=31  lrTauF=0  lrKd=0  lrMVMax=60
测试: 100→400 一发
预期: 上升 ~2.7s / 超调 ≈0° / 收敛 ~5.4s   (允差 ±20%)

目的不是性能，是验证"表、代码、接线、判读"整条链没装反。不过关就回场A查表，不许继续。 顺带跑一次回放辨识（analysis/replay_identify.py）：把控制器复刻喂实录 PV/MV， 复刻输出与实录 MV 的 RMSE ≤5% 才算"机上实跑 = 你以为的配置"——这一步抓过两次"标注与实跑不符"。

5.2 场C：拨盘场（~50 分钟，产出配方 kd 表）

再说一遍这个自造词：拨盘场 = 在机器上给 lrKd 定值的流程。全场只动 lrKd 这一个数 （"刻度盘"），每类目标温度打几发短阶跃，按"超了加、爬了减"的单调规则拨到位。 7-19 的实战记录：三个温度类共 9 发、53 分钟全部收敛——这就是一场标准拨盘场的规模。

步骤内容
1换第一代基础卡（§3.1），lrKd 从理论初值起（400 类从 0.85 起，200 类从 1.1 起，440 类从 1.05 起）
2每个靶点类打 3 发短阶跃：kd = {初值−0.1，初值，初值+0.1}。每发前起点稳 ≥15s
3按拨盘规则收敛：首超 >4° → kd +0.05；不碰线爬行 → kd −0.05。kd 含义单一（刹车松紧），规则单调，一般 1~2 轮收敛
4把收敛值填进配方表。新靶点（如 350）可在相邻类之间插值起步，一发验证
拨盘曲线的形态（知道长相才不慌）：冲透区（到达平坦 1.4~1.7s，首超随 kd 递减）→ 悬崖（kd 差 0.05 在"冲透/爬行"间翻面）→ 爬行区（3s+，首超≈0）。悬崖位置随炉体状态日内游走 ±0.05~0.1， 所以生产值要选在悬崖的安全一侧留一格。同参数两发结果翻面 = 你正踩在悬崖上，退 0.05。

6. 生产 SOP 与维护

项内容
日常配方带 kd 列自动下发；操作工零操作。就两条规矩：①冷机第一炉先热机一次（冷炉体/冷元件的首炉多冲 2~4°，任何控制器同病，热机即愈）；②阶跃前起点稳 ≥15s
免费炉态仪表各温度的保温 MV 顺手记录：它是墙温/日漂的直接读数（AIC9：保 400 从 16.5% 到 22% 就是冷热墙的差）
老化仪表盘稳态锚点（保温 MV ↔ 真保温功率）漂移 <10% 不动；10~20% 重标 q 表（场A 半场）；>20% 全重标
换硬件重标范围换加热元件：q 表 + τe + 拨盘复核（半天）；换功率调节器：先 15 分钟甄别场（保温 MV 带 / 断电初降速对照组 / 90% 与 22% 斜率），判"没变/整体偏移/形状变"再定范围——AIC9 实例：换调节器后增益 0.82×、θ 减半（nDeadSteps 12→6），处方 = 表乘系数 + 4 点复核 + 拨盘抽查 ≈1h；换整机：全流程重走（~1.5 天）。数据谱系纪律：换硬件当天起新旧数据分家标记，永不混用

7. 已知边界（诚实条款——写给三个月后的自己）

边界内容
硬定理"任何日子任何状态都 1.5s + ≤3°"没有控制器能承诺：状态里总有冲刺前测不到的部分（冷炉体首炉、深冷元件、刚换的硬件）×只热不冷的单向执行器 = 尾部风险。第一代的承诺是"暖机典型日达标 + 坏日子多 2~5° + 失败方向偏慢/偏超可控"
200 靶点 + 热炉体墙温高于靶温时免费刹车变负（墙在推），首超 5°+ 是物理不是整定问题。要么等炉体冷、要么该配方 kd 加档、要么接受
物理地板暖机 100→400 理想开关极限 ~1.5s（元件蓄热滑行段刹不掉）。1.71s 距地板仅 0.2s，继续压榨的空间极小
