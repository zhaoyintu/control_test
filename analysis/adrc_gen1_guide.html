ADRC 温控第一代（[1]~[9]）完备指南
参数辨识、代码与上机路径 —— 按本指南操作可复现：100→400 到达 1.5~1.7s / 首超 ≤3°（暖机典型日）

适用对象：AIC9 同类电加热炉（单加热区、单测温点、只加热不制冷、纯滞后 + 一阶执行器）。 本指南自足：从零开始，走完三场标定 + 一次上机爬梯，即达到第一代（代码改动 [1]~[9]）的生产效果。 所有数字以 AIC9 通道 1 的实测为工作示例；换机器时数字全部重测，流程与代码原样复用。

0. 最终效果与总览

工况实测表现（暖机典型日）
100→400°C到达 1.5~1.7s，首超 ≤3°（实测 1.71s / 1.0°），±3° 带内无后续振荡
200→440°C到达 1.4~1.6s，首超 3~4°
100→200°C到达 ~1.0s，首超 ~2.6°（冷炉体）

1. 系统全景：控制器结构与参数分类

1.1 控制链路（代码 = adrc_map_enabled.st，改动 [1]~[9]）

SV ──TD参考(lrWr)──► rv1
                         │
PV ──► ESO(lrWo): z1(干净的PV估计), z2(总扰动=散热+漂移+一切未建模项)
                         │
控制律:  vc = lrWc·(rv1 − z1) − z2 − lrKd·v̂        ← [9] v̂ = vd+z2 (ESO速度估计, 免微分)
                         │
vc(想要的加热速率 °C/s) ──反查 q(u) 表──► MV%      ← Hammerstein 线性化, 环路增益恒 1
                         │
记账: FIFO(nDeadSteps 拍纯滞后) → lrTauF 一阶滤波(元件惯性[7]) → 高温缩水修正[8]
      = ESO 心里的"已喂进多少热量"账本

1.2 参数分两个家族（本指南的核心纪律）

家族成员怎么定性质
模型参数

行为参数

lrK 为什么恒为 1：开表后控制器输出的是"要多少 °C/s 加热速率"，反查表负责兑现—— "1 单位指令 = 1 单位加热速率"按构造成立（量纲：°C/s ÷ °C/s = 1）。增益知识全部住在表里； 拧 K 调速与 wc/wr 简并，且让 ESO 每拍记假账（冲刺攒幻影 → 刹车吐账超调）。调速只拧 wr/wc/kd。

2.2 判读（分析侧脚本，产出可直接粘贴的 ST 参数）

产出方法与工具埋坑警示（都是踩过的）
q∞ 表（aMapU/aMapV）整段拟合分离 q∞/τe/θ：analysis/fit_qinf_0715.py（参考实现）；粗判 theta_from_bump.py 必须整段拟合取稳态马力 q∞——短窗斜率会把元件惯性 τe 混进去，高段低估 ~40%。 MV 重采样必须前值保持（线性插值会把阶跃劈成斜坡导致漏检）。
θ → nDeadSteps阶跃后 PV 起动延迟；nDeadSteps = θ / 10ms θ 对拟合残差是阶梯函数（梯度为零），最小二乘动不了它——必须网格扫描。 取小幅上行值（下行 θ 更长，元件放热；大电平 θ 偏短，辐射耦合）。 注意 θ 里含功率调节器的贡献：AIC9 换调节器后 0.12→0.06s（nDeadSteps 12→6）。
τe → lrTauF整段拟合的一阶惯性项：analysis/fit_taue.py（θ 网格 + 内层最小二乘） 拟合窗 ≤4s（降到底的段会拟出垃圾）；τe 是元件质量属性，350~500°C 平坦（AIC9 实测 0.24~0.34）。
真保温功率（诚实性检查）从稳态保温切断电源后 1.5~3s 的初始降速 = 该保温 MV 的真实功率。 与表值对照可暴露"表 × 散热模型互相抵账"的虚增（AIC9 实测中低段表值虚增 2.03×） 闭环内这种虚增会自我抵消、不影响第一代效果——但它是老化监控的真锚点，务必记录在案。
高温缩水 → lrKqHot/lrKqTref90% 满幅在 >350°C 段的末段斜率 vs 表值：AIC9 实测 −15~25%，台阶饱和型（350°C 后平坦） 不修记账则冲刺期 z2 记负账 → 刹车吐账超调（440 实测 10° 的元凶）。只修记账不修反查：估错方向必为"少给油早刹车"（安全侧）。

200 类  → lrKd := 1.1
400 类  → lrKd := 0.85
440/450 → lrKd := 1.05  (求稳 1.1)
为什么 kd 按配方下发而不是一个数：kd 补的是"理论阻尼需求 − 炉子自带的免费刹车"， 而免费刹车 = 耦合系数 × (腔温 − 墙温)，从 200°C 靶点到 440°C 靶点变化 5 倍以上。 这不是控制器缺陷，是把"每类工况的刹车预算"作为工艺数据放进配方——与商用温控器"多组 PID 参数"同机制， 但每组只有一个数、且有单调拨盘规则。系统本来就按配方下发 SV，kd 只是同表多一列，控制器内无任何 if-else。

4. 理论：行为参数的初值从哪来

把已测的滞后并入，环路近似二阶系统，特征方程 τ·s² + (1+kd)·s + wc = 0，其中 τ = θ + τe（AIC9 v1: 0.36s）：

设计量公式AIC9 数值例
自然频率（决定到达快慢）ωn = √(wc/τ)√(4.5/0.36) ≈ 3.5 rad/s → 300° 阶跃穿越 ~1.5s
阻尼比（决定超调）ζ = (1+kd)/(2√(wc·τ))目标 ζ≈0.8~0.85 → kd 理论初值 ≈ 1.0~1.1
wr（参考快慢）wr ≫ ωn 即为"阶跃参考"30（再大无区别；wr 小时 TD 平滑本身兼职阻尼，就回到低速档形态）
wo（观测器带宽）(2~4)×ωn 带内取10

kd 实测值（0.85~1.1）落在理论初值（1.0~1.1）与孪生折价值之间——因为炉子自带的免费刹车抵了一部分理论阻尼需求， 而免费刹车随靶点/炉态变。所以理论只给初值和方向，最终数值必须由场C 在机器上收尾——这是"认证仪器是机器"原则。

为什么去掉 SV 平滑后必须有 [9]：wr 小的时候参考慢慢走，环路准静态工作，平滑在兼职做阻尼； wr=30 后控制律退化为"比例 + 0.36s 执行器滞后"= 欠阻尼二阶，wc 越大 ζ 越小 （wc=1.55 首超 ~8°，wc=4 无阻尼可荡 40°）。lrKd 用 ESO 的速度估计 v̂ = b0·vd + z2 补回阻尼—— 这就是 PID 里 D 项的角色，但 v̂ 来自模型不对 PV 求导，无噪声放大。

5. 场B + 场C：上机爬梯与拨盘

5.1 场B：第1档回归（15 分钟，烧录后的看门关卡）

参数: lrWc=1.0  lrWo=12  lrWr=3.0  nDeadSteps=31  lrTauF=0  lrKd=0  lrMVMax=60
测试: 100→400 一发
预期: 上升 ~2.7s / 超调 ≈0° / 收敛 ~5.4s   (允差 ±20%)

目的不是性能，是验证"表、代码、接线、判读"整条链没装反。不过关就回场A查表，不许继续。 顺带跑一次回放辨识（analysis/replay_identify.py）：把控制器复刻喂实录 PV/MV， 复刻输出与实录 MV 的 RMSE ≤5% 才算"机上实跑 = 你以为的配置"——这一步抓过两次"标注与实跑不符"。

5.2 场C：拨盘场（~50 分钟，产出配方 kd 表）

步骤内容
1换第一代基础卡（§3.1），lrKd 从理论初值起（400 类从 0.85 起，200 类从 1.1 起，440 类从 1.05 起）
2每个靶点类打 3 发短阶跃：kd = {初值−0.1，初值，初值+0.1}。每发前起点稳 ≥15s
3按拨盘规则收敛：首超 >4° → kd +0.05；不碰线爬行 → kd −0.05。kd 含义单一（刹车松紧），规则单调，一般 1~2 轮收敛
4把收敛值填进配方表。新靶点（如 350）可在相邻类之间插值起步，一发验证
拨盘曲线的形态（知道长相才不慌）：冲透区（到达平坦 1.4~1.7s，首超随 kd 递减）→ 悬崖（kd 差 0.05 在"冲透/爬行"间翻面）→ 爬行区（3s+，首超≈0）。悬崖位置随炉体状态日内游走 ±0.05~0.1， 所以生产值要选在悬崖的安全一侧留一格。同参数两发结果翻面 = 你正踩在悬崖上，退 0.05。

6. 生产 SOP 与维护

项内容
日常配方带 kd 列自动下发；操作工零操作。就两条规矩：①冷机第一炉先热机一次（冷炉体/冷元件的首炉多冲 2~4°，任何控制器同病，热机即愈）；②阶跃前起点稳 ≥15s
免费炉态仪表各温度的保温 MV 顺手记录：它是墙温/日漂的直接读数（AIC9：保 400 从 16.5% 到 22% 就是冷热墙的差）
老化仪表盘稳态锚点（保温 MV ↔ 真保温功率）漂移 <10% 不动；10~20% 重标 q 表（场A 半场）；>20% 全重标
换硬件重标范围换加热元件：q 表 + τe + 拨盘复核（半天）；换功率调节器：先 15 分钟甄别场（保温 MV 带 / 断电初降速对照组 / 90% 与 22% 斜率），判"没变/整体偏移/形状变"再定范围——AIC9 实例：换调节器后增益 0.82×、θ 减半（nDeadSteps 12→6），处方 = 表乘系数 + 4 点复核 + 拨盘抽查 ≈1h；换整机：全流程重走（~1.5 天）。数据谱系纪律：换硬件当天起新旧数据分家标记，永不混用

7. 已知边界（诚实条款——写给三个月后的自己）

边界内容
硬定理"任何日子任何状态都 1.5s + ≤3°"没有控制器能承诺：状态里总有冲刺前测不到的部分（冷炉体首炉、深冷元件、刚换的硬件）×只热不冷的单向执行器 = 尾部风险。第一代的承诺是"暖机典型日达标 + 坏日子多 2~5° + 失败方向偏慢/偏超可控"
200 靶点 + 热炉体墙温高于靶温时免费刹车变负（墙在推），首超 5°+ 是物理不是整定问题。要么等炉体冷、要么该配方 kd 加档、要么接受
物理地板暖机 100→400 理想开关极限 ~1.5s（元件蓄热滑行段刹不掉）。1.71s 距地板仅 0.2s，继续压榨的空间极小
[10] 封存件零表格的单一律（刹车预算 B̂=−z2 在线读）已设计、已包络认证、代码已入库默认关。价签：无在线信息时 4s 级；接通元件状态+机上拨盘后预期回到 1.5~1.7s。触发条件（多机队/规格收紧/工况爆表）出现前不启用
